# Ecosystem State: x402 Protocol

**Last Updated:** 2026-03-25

## Current Version

| Property | Value |
|----------|-------|
| Protocol version | v2 |
| Foundation | Established (Coinbase + Cloudflare, Dec 2025) |
| SDKs | @x402/core, @x402/evm, @x402/svm, @x402/fetch, @x402/express |
| Other SDKs | Go, Python also available |
| Free tier | 1,000 tx/month |

## Key Notes

- **Permit2 support:** x402 v2 now supports ALL ERC-20 tokens via Permit2,
  not just EIP-3009 tokens. This means we could support tokens beyond USDC
  in Phase 4 without protocol changes.
- **Google AP2 integration:** Google's Agent-to-Pay protocol integrates x402.
  Could be relevant for external agent onboarding.
- **Base native:** x402 is designed for Base L2. Sub-cent fees make
  micropayments viable for agent-to-agent settlement.

## Our Usage

- AgentWallet: `executeX402Payment()` for USDC settlement
- Phase 1: agent x402 payments on-chain (agents need real-time settlement from day one)
- Phase 4: x402 rails open to external AI agents

## Integration Notes

```typescript
// TypeScript x402 client pattern
import { createX402Client } from '@x402/fetch';

const client = createX402Client({
  chain: 'base',
  wallet: agentWallet,
});

// HTTP-native payment
const response = await client.fetch('https://service.example/api', {
  maxPayment: { amount: '1.00', currency: 'USDC' },
});
```
